<p>
    Each upload creates a new stream that is based on a pre-defined stream configuration template.
</p>
<p>
    The configuration templates dictates how a stream is structured and how the data is processed.
</p>
<p>
    You can find below the pre-defined stream configuration templates that are available for use.
</p>

<AccordionGroup>
    <Accordion title="Transcribe">
        Stream Configuration Template Id: `65f169a51351291e67a7a7fa`<br/>
        Input data: `wav` `mp3` `mp4` `m4a` `webm` etc.<br/>
        Output data: `partial and final transcription packets`<br/>
    </Accordion>
    <Accordion title="Diarize speakers">
        Stream Configuration Template Id: `665c1e117d091e7572d3cb19`<br/>
        Input data: `wav` `mp3` `mp4` `m4a` `webm` etc.<br/>
        Output data: `partial and final diarization packets`<br/>
    </Accordion>
    <Accordion title="Transcribe with diarization">
        Stream Configuration Template Id: `667a9e9485262d98894fe51b`<br/>
        Input data: `wav` `mp3` `mp4` `m4a` `webm` etc.<br/>
        Output data: `partial and final transcription and diarization packets`<br/>
    </Accordion>
    <Accordion title="File-optimized transcribe">
        Stream Configuration Template Id: `668115d123bca7e3509723d4`<br/>
        Input data: `wav` `mp3` `mp4` `m4a` `webm` etc.<br/>
        Output data: `final transcription packets`<br/>

        | Query parameter | Default value | Description |
        | --- | --- | --- |
        | `language` | None | The language as country ISO code to transcribe the audio to. When not specified, it is automatically detected |
        | `noSpeechThreshold` | 0.5 | The threshold value to consider a segment as a speech segment. The value ranges from 0 to 1 |

    </Accordion>
</AccordionGroup>
